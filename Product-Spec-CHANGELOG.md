## 1.6.0 - 2026-06-09

### 变更类型：修改

### 修改功能
- **功能 4**：快照指纹去重，相同行情跳过 LLM；`temperature: 0`
- 休市多次刷新结论保持一致；Footer「行情未变·沿用」
- ADR-003 休市诊断一致性

### 影响范围
- LLM 模块：高 — 指纹缓存 + 跳过逻辑
- 用户体验：休市刷新不再「结论乱跳」

### 兼容性说明
- 旧诊断缓存无指纹；下次新诊断后写入指纹

## 1.5.0 - 2026-06-09

### 变更类型：修改

### 修改功能
- 自选池由「5 只合计」改为 **股票 5 + 场内基金 5 分池计数**（合计最多 10 只）
- 功能 2：腾讯 JSONP 每批 5 只，10 只时分 2 批
- 功能 5 / UI-Spec：双区布局 `股票监控 (n/5)` + `基金监控 (n/5)`
- 性能指标：单轮刷新 ≤6s 行情 / ≤120s AI
- ADR-002 更新为用户确认的分池方案

### 影响范围
- 配置与 UI：高 — 分池校验与双区展示
- 行情/调度：中 — 批量请求拆批
- LLM：中 — 最多 10 次串行调用

### 兼容性说明
- 旧 watchlist 按代码自动推断 `instrumentType` 并归入对应池
- 若旧数据超过单池 5 只，保留前 5 只并 Toast 提示（开发实现）

- 功能 2/4：输入输出由「股票」统一为「证券」
- 项目定位：A 股 + 场内基金实时诊断

### 删除功能
- 无

### 功能调整
- 自选池：仍为 5 只合计（未采用股票 5 + 基金 5 各算方案）

### 影响范围
- 代码识别模块：高 — 需扩展 `normalizeStockCode`
- 行情/LLM/UI：中 — 字段与文案扩展，JSONP 路径不变
- 调度模块：低 — 场内基金沿用 A 股交易时段

### 兼容性说明
- 向后兼容：旧 watchlist 无 `instrumentType` 时默认 `stock`
- 迁移指南：无需用户操作

## 1.2.0 - 2026-06-09

### 变更类型：新增

### 新增功能
- **功能 6：交易日历同步** — 每年 1 月 1 日及首次缺失时自动拉取当年国务院节假日数据
- 主源 holiday-cn（JSDelivr CDN），备源 timor.tech
- `isTradingDay()` 判定：支持法定放假与调休补班日
- 配置区展示日历同步状态 + 手动「重新同步日历」兜底

### 修改功能
- **功能 3 定时调度**：交易日判定从「周一至周五」升级为「日历优先 +  weekday 降级」
- 删除 v1.1.0 中「不含法定休市日历」的临时方案

### 删除功能
- 无

### 功能调整
- **法定休市日历自动识别**：低优先级 → **高优先级**（功能 6）

### 非功能性需求调整
- 性能：交易日历拉取 ≤ 2s，命中缓存零请求
- 安全：允许向 `cdn.jsdelivr.net`、`timor.tech` 发起日历请求

### 技术栈调整
- 新增 `services/tradingCalendar.ts`
- LocalStorage Key：`agu_trading_calendar_{year}`

### 影响范围
- 调度模块：高 — `isTradingDay` 接入所有自动触发逻辑
- 配置区 UI：低 — 新增日历状态展示
- 网络层：低 — 每年仅 1 次主动拉取（缓存命中后无请求）

### 兼容性说明
- 向后兼容：LocalStorage 无日历数据时自动降级为周一至周五，不影响现有流程
- 迁移指南：无需用户操作，App 启动时自动同步

## 1.1.0 - 2026-06-09

### 变更类型：重构

### 新增功能
- 独立 `UI-Spec.md`：UI/UX 规范与 System Prompt 模板
- 独立 `Product-Spec-CHANGELOG.md`：变更记录与版本管理
- 行情源方案：腾讯 JSONP 主源 + 新浪 JSONP 备源，明确禁止 fetch 直连
- 定时对齐规则：09:30 起 30 分钟网格，午休/收盘停止自动调度
- 目标用户、功能优先级、非功能性需求、技术栈等完整章节

### 修改功能
- 数据依据：`30 分钟 K 线` → `实时快照 + 对齐时刻标签`（与用户确认一致）
- API Key 存储：`加密/明文` → **明文** LocalStorage（自用可接受）
- 技术栈：`Vanilla JS 或 Vue3` → **Vue 3 + Vite + TypeScript**
- 设备目标：明确手机 + 桌面响应式兼容
- LLM 调用：5 只串行，间隔 ≥ 300ms；诊断结果持久化

### 删除功能
- 无

### 功能调整
- 行情拉取优先级：高（原 FR-2.1 fetch 直连方案废弃）

### AI增强调整
- System Prompt 改写：强调输入为实时快照而非 K 线序列
- 新增「特殊时段 Prompt 自适应」（集合竞价 / 非交易时段）

### 非功能性需求调整
- 新增性能、安全、可用性、兼容性四类完整指标
- 浏览器兼容列表：Chrome/Edge/Firefox/Safari 最新 2 版本 + 移动端

### 技术栈调整
- 前端框架确定为 Vue 3 Composition API
- 行情接入层：`services/quoteJsonp.ts` JSONP 封装

### 影响范围
- 行情模块：高 — 实现方式从 fetch 改为 JSONP
- 调度模块：中 — 对齐算法需按网格时刻实现
- UI 模块：低 — 规范迁移至 UI-Spec.md，视觉不变

### 兼容性说明
- 向后兼容：文档层面不兼容（v1.0 结构已重构）
- 迁移指南：开发以 Product-Spec.md 1.1.0 + UI-Spec.md 为准
