## 2.4.0 - 2026-06-16

### 变更类型：新增 / 重构（次版本）

### 新增功能
- **自定义分组系统**：
  - `UserGroup` 类型（`id`/`name`/`order`/`collapsed`），存入 `config.groups`
  - 左侧可折叠侧边栏（200px，可拖拽 160–280px，hover 展开，移动端隐藏）
  - 新建/重命名/删除分组（空分组保留，无上限）
  - 分组排序（↑↓ / 拖拽），折叠状态持久化
  - 主页按分组归类，拖拽卡片入组，内建「未分组」
- **三态刷新开关**（替代 v2.2 智能降频）：
  - `WatchlistItem.refreshMode` 枚举：`off` / `normal`（默认）/ `overclock`
  - 卡片 Header ◌/⚡/⚡⚡ 三态切换按钮
  - 超频 = 主调度正常刷新 + 独立 30s 高频行情监控（仅拉行情 + 预警，不触发 AI）
  - 超频上限 5 只，要求已配置至少 1 个关键位
  - 主调度恢复全池刷新（`normal` + `overclock` 的证券均参与）

### 修改功能
- **功能 1**：`WatchlistItem` 新增 `groupId`/`refreshMode`；`AppConfig` 新增 `groups`；废弃 `autoRefresh`
- **功能 3**：主调度恢复全池自动刷新；超频由用户手动控制，与 `refreshFrequency` 解耦
- **功能 5**：新增侧边栏布局 + 主页分组展示 + 卡片三态图标
- **功能 8**：废弃智能降频表；高频监控改为「超频模式」用户自主控制
- **ADR-007**：完全重写，废弃档位感知降频，采用三态开关 + 上限控制

### 删除 / 废弃
- **废弃** 智能降频（按主调度档位自动决定 30s 轮询）
- **废弃** `autoRefresh: boolean` → 改为 `refreshMode: 'off' | 'normal' | 'overclock'`

### 影响范围
- 类型系统：高；调度器：高；UI：高；ADR-007：重写
- 文档：Product-Spec / README / UI-Spec 同步至 2.4.0

### 兼容性说明
- 旧 `autoRefresh: true` → `refreshMode: 'normal'`；`autoRefresh: false` → `refreshMode: 'off'`
- 旧配置无 `groups` → 默认空数组；无 `groupId` → 归入「未分组」

## 2.3.0 - 2026-06-16

### 变更类型：修改（次版本）

### 修改功能
- **自选池上限扩展**：从「5 股 + 5 基分池」改为 **统一自选池最多 20 只**（股票与场内基金混排，不再分池计数）
- **UI 统一化**：移除「股票监控 (n/5)」「基金监控 (n/5)」双区布局，改为单一 `自选监控 (n/20)` Grid
- **悬浮窗钉选**：上限由 4 只扩展为 **5 只**（下限仍为 1 只）
- **行情拉取**：批量请求由 2 批（10 只）扩展为 **4 批（20 只）**
- **LLM 调用**：串行上限由 10 扩展为 20
- **性能指标**：全量刷新行情 ≤ 12s、AI ≤ 240s

### 删除 / 废弃
- **废弃** `MAX_STOCKS` / `MAX_FUNDS` 分池常数，统一为 `MAX_SECURITIES = 20`
- **废弃** Toast 文案中的分池提示（「股票已达上限 5 只」「基金已达上限 5 只」）
- **废弃** 功能 5 双区「先股票后基金」排序规则

### 预告（v2.4）
- 用户自定义分区分组（可创建、命名、删除分组，每组可折叠）
- 每只证券独立「自动刷新」开关（按需控制 Token 消耗）

### 影响范围
- 类型系统：高 — `MAX_STOCKS`/`MAX_FUNDS` → `MAX_SECURITIES`、`widgetPin` max 常量变更
- UI：高 — 移除双区布局，统一为单区 Grid；i18n 文案同步
- 行情/LLM：中 — 批量请求与串行上限扩展
- 文档：Product-Spec / README / UI-Spec 同步至 2.3.0

### 兼容性说明
- 旧 `WatchlistItem` 保留 `instrumentType`（用于 AI Prompt 区分），但不再用于池计数
- 旧 watchlist 中的股票与基金在原分区迁移时按代码顺序合并，超出 20 只保留前 20 只并 Toast
- 悬浮窗钉选若旧配置超过 5 只，截断至前 5 只

## 2.2.0 - 2026-06-12

### 变更类型：新增（次版本）

### 新增功能
- **功能 8：桌面通知预警系统（Electron 专属）**：
  - **AI 自动关键位计算**：每次诊断 LLM 输出 `supportLevel` / `resistanceLevel`，自动写入证券配置（偏差保护 ±3%）
  - **价格突破预警**：智能降频高频率监控（5/15 分钟档蹭主调度，30/60 分钟档 30s 独立轮询），价格穿透关键位时桌面弹窗
  - **Signal 高风险弹窗**：固定词库（止损 / 逢高减仓 / 减仓 / 清仓）命中时触发
  - **异常通知**：API Key 鉴权失败、行情全部失败时提醒
  - **独立弹窗**：360×200 无边框半透明右下角小卡片，15s 自动消失，多条合并展示
  - **配置开关**：每一种通知类型独立可控，含总开关
- **AI增强 3**：关键位自动计算与预警
- **附录 ADR-007**：智能降频价格监控架构决策

### 修改功能
- **功能 1**：配置对象新增 `alertSettings`；`WatchlistItem` 新增 `keyLevels` / `keyLevelsLocked`
- **功能 4**：`DiagnosisResult` 新增 `supportLevel` / `resistanceLevel` 必填字段；System Prompt 扩展
- **功能 7**：桌面悬浮与通知预警系统联动；新增 IPC 信道 `agu:send-alert`
- **项目概述**：一句话描述补充桌面通知预警能力
- **非功能性需求**：新增高频轮询 ≤ 1s、弹窗 ≤ 300ms 性能指标

### 影响范围
- Desktop 专属：高 — 新增 `alertService.ts`、主进程弹窗 IPC、独立调度器、关键位管理；需重新 `npm run build:main` 并打包
- Web H5：低 — `DiagnosisResult` 扩展字段向后兼容，H5 忽略 `alertSettings` / `keyLevels`（TypeScript 标记可选）
- 文档：Product-Spec / README / UI-Spec（附录 A Prompt 模板）同步至 2.2.0

### 兼容性说明
- 旧 `DiagnosisResult` 无 `supportLevel` / `resistanceLevel`：读取时默认 `null`，不参与关键位判定
- 旧 `WatchlistItem` 无 `keyLevels`：新增字段可选，不影响现有自选数据
- 旧配置无 `alertSettings`：默认全部启用
- H5 浏览器版不受影响，通知功能静默跳过
