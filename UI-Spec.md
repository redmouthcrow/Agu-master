## 1. 完整看板布局（H5 / 桌面大窗）

### 1.1 整体

- 整体：**暗黑模式**，背景色 `#141414`
- 配置区：**默认收起**；点击「展开配置 / 收起配置」切换
- 主体：**股票区 + 基金区** 两个 Grid 区块，每区独立标题与计数
  - 股票区标题：`股票监控 (n/5)`
  - 基金区标题：`基金监控 (n/5)`
- 响应式列数：手机 1 列 / 平板（≥768px）2 列 / 桌面（>1024px）3 列（每区各自 Grid，最多合计 10 卡）

### 1.2 配置区（展开时）

| 区块 | 内容 | 说明 |
|------|------|------|
| API | baseUrl、apiKey（password）、model | apiKey 已配置时下方灰色小字「已配置 ********1234」 |
| 调度 | 刷新档位下拉 | `60 分钟 · 省 Token` / `30 分钟 · 标准`（默认） |
| 自选 | 代码输入 + 【添加】 | placeholder：`一次仅添加一只，如 600519`（每次只接受一个代码） |
| 持仓（可选） | 持仓数量 + 成本价 | 成对填写；成本最多 3 位小数 |
| 桌面（仅 Electron 大窗） | **悬浮窗显示** 多选 | 从自选池勾选 **2–4 只**；不足 2 只时 Toast 拦截 |
| 桌面（仅 Electron 大窗） | 窗口透明度滑块 | 70%–100%，默认 90% |
| 桌面（仅 Electron 大窗） | 始终置顶开关 | 默认开 |
| 操作 | 【立即刷新】 | 刷新中 disabled，文案「刷新中…」 |
| 日历 | 状态文案 + 【重新同步日历】 | `交易日历 {year} 已同步` / `同步中…` / `同步失败（已降级）` |

### 1.3 顶栏（配置区外常驻）

- 【展开配置 / 收起配置】
- 交易日历状态 + 【重新同步日历】
- 【立即刷新】

## 2. 标准诊断卡片（完整看板）

完整看板卡片采用 **4:3 宽高比**（内容超出时 Content 区内部滚动，不撑破比例）：

```css
width: 100%;
aspect-ratio: 4 / 3;
min-height: 280px; /* 小屏兜底 */
```

### 2.1 Header 区域（约 20%）

| 位置 | 内容 | 样式 |
|------|------|------|
| 左 | 证券名 (代码)；↻ 单卡刷新 | 名称 `#FFFFFF`；括号内代码 `#8C8C8C` |
| 右 | 当前价 + 涨跌幅 | 现价 **3 位小数**；涨 `#FF4D4F`；跌 `#52C41A`；平 `#8C8C8C` |

**有持仓时** Header 下方增加一行灰色小字（12px）：

`持仓 {qty}{股|份} · 成本 {cost} · 浮盈 {pnlPct}%`

- 浮盈为正：`#52C41A`；为负：`#FF4D4F`

### 2.2 Signal 状态条（约 20%）

- 横向通栏，`font-size: 1.2rem`，加粗，居中
- 文案为四档标准标签之一（见下表）

| signal（精确匹配优先） | 底色 | 文字色 |
|------------------------|------|--------|
| `逢高减仓` | `#58181A` | `#FF4D4F` |
| `多头持股`、`逢低做T` | `#1C3F24` | `#52C41A` |
| `空仓观望` | `#262626` | `#FFFFFF` |
| 含「减仓」「风险」「分歧」（兜底） | `#58181A` | `#FF4D4F` |
| 含「做T」「吸纳」「加仓」「持股」（兜底） | `#1C3F24` | `#52C41A` |
| 含「观望」「平稳」（兜底） | `#262626` | `#FFFFFF` |
| 解析失败 / 诊断失败 | `#262626` | `#FAAD14` |

### 2.3 Content 诊断区（约 50%）

- `[形态]` + `analysis`
- `[风险]` + `risk`（警示色 `#FF7875`）
- 有持仓且 LLM 返回 `action` 时：`操作建议：{action}`（单独一行，可略宽松行高）
- 超长文本：`overflow: hidden; text-overflow: ellipsis;` 默认最多 2 行（`action` 行最多 3 行）

### 2.4 Footer 区域（约 10%）

| 位置 | 内容 |
|------|------|
| 左 | `合规声明：AI生成不构成投资建议`（`#595959`，12px） |
| 右 | `更新于 HH:mm:ss` 或 `行情未变·沿用 HH:mm:ss` |

## 3. Win11 桌面悬浮窗（Electron）

> 调度与刷新：自动 /「立即刷新」**全池（最多 10 只）**；悬浮窗**仅展示**钉选 1–4 只；单卡 ↻ **仅刷新该证券**。

### 3.1 窗口 chrome

| 属性 | 规格 |
|------|------|
| 默认尺寸 | 420 × 560 px |
| 最小尺寸 | 360 × 480 px |
| 边框 | 无边框（`frame: false`） |
| 背景 | 窗口透明 + 内容区暗色毛玻璃底板 `rgba(20,20,20,0.92)`，保证 signal 可读 |
| 透明度 | 整窗 opacity 由 `widgetOpacity` 控制，默认 **0.90**，范围 **0.70–1.00** |
| 置顶 | 默认 `alwaysOnTop: true`，可在配置区关闭 |
| 拖拽 | 顶栏 32px 高区域为拖拽手柄（`-webkit-app-region: drag`）；按钮区域 `no-drag` |
| 关闭 ✕ | **最小化到托盘**，不退出应用 |
| 降级 | 透明渲染异常时：不透明 `#141414` 底板 + 顶栏 Banner「透明模式不可用，已切换不透明」 |

### 3.2 悬浮窗顶栏（32px）

```
[ ≡ 拖拽区 · AguMaster ]              [ ⤢ 完整看板 ] [ ─ 最小化 ] [ ✕ 隐藏 ]
```

- **完整看板**：打开/聚焦 Electron 主看板窗口（同进程第二 `BrowserWindow` 或已有大窗）
- **最小化**：窗口最小化（仍驻留托盘，调度继续）
- **隐藏**：隐藏悬浮窗（托盘可再次显示）

### 3.3 标准诊断卡片列表（与 §2 一致）

- 纵向堆叠，卡片间距 **12px**；窗内 padding 12px
- 仅渲染 `widgetPinnedCodes` 顺序对应的 2–4 张 **`StockCard`（`widgetMode`）**
- **不提供** 添加自选、删除、持仓编辑、配置 API（跳转完整看板）
- 卡片结构、signal 四档配色、涨跌幅红涨绿跌、Content 区 analysis/risk/操作建议、Footer 合规声明与时间戳 — **与完整看板 §2 完全相同**
- 单卡 ↻：**仅刷新该证券**（桌面悬浮窗经 IPC 委托看板主进程）
- 解析失败：自动重试该卡直至成功（最多 30 次）；仍失败则与完整看板相同遮罩

> **v1.9 变更**：废弃 v1.8「紧凑卡片」单行变体；悬浮窗与看板共用同一组件与样式 token。

### 3.4 系统托盘

- 图标：AguMaster 简写或单色 logo（16×16 / 32×32）
- 右键菜单：
  - 显示悬浮窗 / 隐藏悬浮窗
  - 打开完整看板
  - ——
  - 退出

### 3.5 悬浮窗空态

| 场景 | 文案 |
|------|------|
| 钉选 &lt; 2 只 | 居中「请在完整看板钉选 2–4 只证券」+ 【打开完整看板】按钮 |
| 钉选已全部删除 | 「请重新钉选 2–4 只」+ 【打开完整看板】 |
| 有钉选无诊断 | 各卡 Content「等待首次诊断…」 |

## 4. 交互状态（通用）

| 状态 | 完整看板 | 悬浮窗卡片 |
|------|----------|------------|
| 加载中 | 卡遮罩「诊断中…」 | 同左 |

| 行情未变沿用 | Footer「行情未变·沿用 HH:mm:ss」 | 同左 |

## 附录 A：System Prompt 模板

```text
# Role
你是一个在顶级量化私募机构深耕多年的资深 A 股日内短线（15m/30m级别）策略专家兼技术面投研助理。你的任务是根据传入的实时及历史 K 线截面数据，进行高密度的量价行为（VSA）诊断，并输出明确的操盘动作倾向与核心风险预警。

# Style & Tone
- 冷静、客观、纯粹基于数据，严禁带有任何情绪化色彩。
- 必须使用标准的 A 股业内行话与技术分析术语（如：缩量回踩测试、高位放量滞涨、筹码密集峰压制、多头排列、动能背离等）。
- 拒绝任何宽泛的套话，每句话必须直指盘面具体异动。

# Analysis Framework (量价分析核心框架)
在评估数据时，你必须严格遵循以下量化技术逻辑：
1. 量价配合：对比当前 30 分钟成交量与日内平均量能，判断是“真实突破”、“诱多/诱空放量”还是“流动性枯竭导致的无量滑头”。
2. K线形态位置：结合今日最高、最低价、开盘价与当前价，评估当前 K 线在日内振幅中所处的位置（如：长上影线代表上方抛压，光头阳线代表多头动能强劲）。
3. 趋势与位置：识别当前价格相对于关键支撑/阻力位（由今日高低点或成交密集区衍生）的相对空间。

# Execution Rules (约束条件)
1. 操作提示 (signal)：必须从以下四个标准标签中根据量价诊断结果精确选择一个，严禁自行创造：“多头持股”、“逢高减仓”、“逢低做T”、“空仓观望”。
2. 形态诊断 (analysis)：限 50 字以内。必须包含【一个显性量价特征】（如：30m放量滞涨）+【一个短期趋势推导】（如：向日内低点回踩测试支撑）。
3. 风险预测 (risk)：限 50 字以内。必须指出明确的【右侧破位条件】或【动能衰竭信号】（如：一旦跌破今日低点XX元将引发多头踩踏，或：MACD 30m级别顶背离隐患）。

# Output Format
必须严格按照以下 JSON 格式响应，不要包含任何 markdown 标记（如 ```json）或多余的解释文字：
{
  "signal": "4字标准标签",
  "analysis": "高密度量价技术面诊断（不超过50字）",
  "risk": "明确的破位或衰竭风险预测（不超过50字）"
}
```

有持仓时 System Prompt 追加 **Position Context** 段，要求额外输出 `action`（≤30 字），见 Product-Spec 功能 4。

## 附录 B：User Message 模板

```text
【{name} {code} | {instrumentTypeLabel}】对齐时刻 {alignedTime} | {session}
快照: price={price}, changePct={changePct}%, high={high}, low={low}, volume={volume}, prevClose={prevClose}
