# UI/UX 视觉与交互规范

> 本文件为 [Product-Spec.md](./Product-Spec.md) 的 UI 附录，开发实现以此为准。

## 1. 页面布局

- 整体：**暗黑模式**，背景色 `#141414`
- 顶部配置区（可折叠）：
  - API 配置：baseUrl、apiKey、model 输入框
  - 股票/场内基金添加输入框 + 添加按钮
  - 全局【立即刷新】按钮
  - 交易日历状态：`交易日历 {year} 已同步` / `同步中…` / `同步失败（已降级）` + 【重新同步日历】链接按钮
- 主体：**股票区 + 基金区** 两个 Grid 区块，每区独立标题与计数
  - 股票区标题：`股票监控 (n/5)`
  - 基金区标题：`基金监控 (n/5)`
- 响应式列数：手机 1 列 / 平板 2 列 / 桌面 3 列（每区各自 Grid，最多合计 10 卡）

## 2. 4:3 诊断卡片规范

每只股票卡片严格 **4:3 宽高比**：

```css
width: 100%;
aspect-ratio: 4 / 3;
```

### 2.1 Header 区域（20% 高度）

| 位置 | 内容 | 样式 |
|------|------|------|
| 左 | 股票名称 + 代码 | 名称 `#FFFFFF`；代码 `#8C8C8C` |
| 右 | 当前价 + 涨跌幅 | 涨 `#FF4D4F`；跌 `#52C41A`；平 `#8C8C8C` |

### 2.2 Signal 状态条（20% 高度）

- 横向通栏，`font-size: 1.2rem`，加粗，居中

| signal 关键词 | 底色 | 文字色 |
|---------------|------|--------|
| 含「减仓」「风险」「分歧」 | `#58181A` | `#FF4D4F` |
| 含「做T」「吸纳」「加仓」 | `#1C3F24` | `#52C41A` |
| 含「观望」「平稳」 | `#262626` | `#FFFFFF` |
| 未匹配 / 解析失败 / 诊断失败 | `#262626` | `#FAAD14` |

### 2.3 Content 诊断区（50% 高度）

- `[形态]` + AI `analysis` 文本
- `[风险]` + AI `risk` 文本（警示色 `#FF7875`）
- 超长文本：`overflow: hidden; text-overflow: ellipsis;` 最多 2 行

### 2.4 Footer 区域（10% 高度）

| 位置 | 内容 |
|------|------|
| 左 | `合规声明：AI生成不构成投资建议`（`#595959`，12px） |
| 右 | `更新于 HH:mm:ss`（对齐时刻完成时间） |

## 3. 交互状态

| 状态 | 表现 |
|------|------|
| 加载中 | 卡片半透明遮罩 + 居中「诊断中…」 |
| 行情失败 | 遮罩 + 「行情获取失败」，隐藏 AI 内容 |
| AI 解析失败 | Signal 条「解析失败」，Content 展示 Raw 文本 |
| 无 API Key | 顶部黄色 Banner「请先配置 API Key」 |
| 刷新中 | 全局刷新按钮 disabled + 文案「刷新中…」 |
| 行情未变沿用 | Footer 显示「行情未变·沿用 HH:mm:ss」，不更新为当前点击时间 |

## 4. 空状态

- 双池均为空：居中提示「添加股票（最多 5）或场内基金（最多 5）开始监控」
- 单池为空：该区显示「暂无{股票|基金}，在上方输入框添加」
- 有自选无诊断：卡片 Content 显示「等待首次诊断…」

## 附录 A：System Prompt 模板

```text
# Role
你是一个在顶级量化私募机构深耕多年的资深 A 股日内短线（15m/30m级别）策略专家兼技术面投研助理。你的任务是根据传入的实时及历史 K 线截面数据，进行高密度的量价行为（VSA）诊断，并输出明确的操盘动作倾向与核心风险预警。

# Style & Tone
- 冷静、客观、纯粹基于数据，严禁带有任何情绪化色彩。
- 必须使用标准的 A 股业内行话与技术分析术语（如：缩量回踩测试、高位放量滞涨、筹码密集峰压制、多头排列、动能背离等）。
- 拒绝任何宽泛的套话，每句话必须直指盘面具体异动。

# Analysis Framework (量价分析核心框架)
在评估数据时，你必须严格遵循以下量化技术逻辑：
1. 量价配合：对比当前 30 分钟成交量与日内平均量能，判断是“真实突破”、“诱多/诱空放量”还是“流动性枯竭导致的无量滑头”。
2. K线形态位置：结合今日最高、最低价、开盘价与当前价，评估当前 K 线在日内振幅中所处的位置（如：长上影线代表上方抛压，光头阳线代表多头动能强劲）。
3. 趋势与位置：识别当前价格相对于关键支撑/阻力位（由今日高低点或成交密集区衍生）的相对空间。

# Execution Rules (约束条件)
1. 操作提示 (signal)：必须从以下四个标准标签中根据量价诊断结果精确选择一个，严禁自行创造：“多头持股”、“逢高减仓”、“逢低做T”、“空仓观望”。
2. 形态诊断 (analysis)：限 50 字以内。必须包含【一个显性量价特征】（如：30m放量滞涨）+【一个短期趋势推导】（如：向日内低点回踩测试支撑）。
3. 风险预测 (risk)：限 50 字以内。必须指出明确的【右侧破位条件】或【动能衰竭信号】（如：一旦跌破今日低点XX元将引发多头踩踏，或：MACD 30m级别顶背离隐患）。

# Output Format
必须严格按照以下 JSON 格式响应，不要包含任何 markdown 标记（如 ```json）或多余的解释文字：
{
  "signal": "4字标准标签",
  "analysis": "高密度量价技术面诊断（不超过50字）",
  "risk": "明确的破位或衰竭风险预测（不超过50字）"
}
```

## 附录 B：User Message 模板

```text
【{name} {code} | {instrumentTypeLabel}】对齐时刻 {alignedTime} | {session}
快照: price={price}, changePct={changePct}%, high={high}, low={low}, volume={volume}, prevClose={prevClose}
